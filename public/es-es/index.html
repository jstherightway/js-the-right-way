PRIMEROS PASOS

ACERCA

Creado por Netscape en 1995 como una extensión de HTML para Netscape Navigator 2.0, JavaScript ha tenido como función principal la manipulación de los documentos HTML y la validación de formularios. Antes de ganar su nombre muy famoso en estos días, JavaScript fue llamado Mocha. Cuando fue lanzado su primera publicación beta, fue oficialmente llamado LiveScript y finalmente, cuando fue publicado por Sun Microsystems, fue bautizado con el nombre por el cual es conocido hoy. A causa de los nombres similares, las personas confunden JavaScript con Java. Aunque ambos tienen la estructura léxica de programación, ellos no son el mismo lenguaje. Diferente de C, C# y Java, JavaScript es un lenguaje interpretado. Significa que este requiere un "interpreter". En el caso de JavaScript, el interpreter es el navegador.

VERSION ACTUAL

El estándar de JavaScript es ECMAScript. A partir del 2012, todos los navegadores modernos soportan completamente ECMAScript 5.1. Viejos navegadores soportan al menos ECMAScript 3. La 6ta. revisión importante del estándar se está trabajando.

Una buena referencia de versiones, gu;ias y noticias acerca de JavaScript pueden encontrarse en la Read de Desarrolladores Mozilla.

EL DOM

El Modelo de Objetos del Documento (DOM) es un API para documentos HTML y XML. Este prevee una representación estructural del documento, permitiéndole modificar su contenido y presentación visual mediante el uso de un lenguaje de scripting tal como JavaScript. Ver mas en Red de Desarrolladores Mozilla - DOM.

LAS BUENAS PARTES

Orientado a Objetos

JavaScript tiene una fuerte capacidad de programación orientada a objetos, a pesar de algunos debates que tienen lugar debido a las diferencias de la orientacion a objetos en Javascript comparado con otros lenguajes.

Fuente: Introducción a JavaScript Orientado a Objetos

Funciones Anónimas

Funciones anómimas que son declaradas dinámicamente en tiempo de ejecución. Ellas son llamadas anónimas porque ellas no tienen un nombre de la misma manera que las funciones normales.

Fuente: Funciones anónimas

Funciones de Objetos de primera clase

Las funciones en JavaScript son objetos de primera clase. Esto significa que las funciones JavaScript son simplemente un tipo de objeto especial que pueden hacer todas las cosas que un objeto regular puede hacer.

Fuente: Las funciones son objetos de primera clase en JavaScript

Tipado Blando

Para muchos desarrolladores front-end, JavaScript fue su primera experiencia un lenguaje de scripting y/o interpretado. Para estos desarrolladores, el concepto y las implicaciones de las variables de tipado blando quizas con de segunda naturaleza. Como siempre, el crecimiento explosivo en demanda para aplicaciones web modernas han resultado en un crecimiento en el número de desarrolladores back-end que han tenido que zambullir sus pies dentro de la piscina de las tecnologías del lado cliente. Muchos de estos programadores llegan con fuerte transfondo de lenguajes de tipado fuerte, tales como C# o Java, y le son desconocidos la libertad y las potenciales trampas envueltas en el trabajo con variables de tipado blando.

Source: Understanding Loose Typing in JavaScript

Alcance y Elevación

Alcance: En JavaScript, las funciones son nuestras delimitadoras del alcance de facto al mometno de la declaración de variables, lo cual significa que los bloques que habitalmente de bucles y condicionales (tales como if, for, while, switch y try) NO delimitan el alcance, a diferencia de la mayoría de otros lenguajes. Por lo tanto, estos bloques compartirán el mismo alcance que la función que as contiene. Esta forma, podría ser peligroso declarar variables dentro de bloques si pudieran ser vistas las variables solo en el bloque que pertenecen.

Elevación: En ejecución, todas las declaraciones var y function son movidas para el principio de la de cada función (su alcance) - esto es conocido como Elevación. Dicho esto, es buena practica declarar todas las variables todas juntas en la primera línea, con el fín de evitar falsas expectativas con las variables que han sido declaradas luego de la asignación de su valor - esto es un problema común con programadores que llegan de lenguajes con alcance de bloques.

Fuentes: Alcance y Elevación en JavaScript

Función de Vinculación

La Función de Vinculación es probablemente la última de las preocupaciones de un principiante con JavaScript, pero cuando se entiende que necesita una solución al problema de como mantener el contexto de 'this' dentro de otra función, entonces puede darse cuenta que de hecho necesita está Function.prototype.bind().

Source: Understanding JavaScript’s Function.prototype.bind

Función de Clausura

Las Clausuras son funciones que hacen referencia a variables (libres) independientes. En otras palabras, la función definida en la clausura 'recuerda' el ambiente en el cual esta fue creado. Esto es un importante concepto para entender, así como útil durante el desarrollo, por ejemplo emular métodos privados. Esto puede también ayuda a aprender como evitar errores, como crear clausuras en bucles.

Fuente: MDN - Clausuras

Modo EstrictoStrict

El modo estricto de ECMAScript 5 es una manera de optar a una variable restringida de JavaScript. El modo estricto no es solo un subconjunto: este tiene intencionalmente un a semántica diferente del código normal. Los navegadores que no soportan el modo estricto correran el código de forma distinta con aquellos navegadores que lo soportan, entonces no confie en el modo estricto sin una prueba de característica que soporte los aspectos relevantes del modo estricto. El código en modo estricto y no estrícto pueden coexistir, de esta forma, scripts pueden optar entrar al modo estrícto de forma incremental. Strict mode code and non-strict mode code can coexist, so scripts can opt into strict mode incrementally.

Source: MDN - Strict mode

Expresión de Función Inmediatamente Invocada (IIFE)

Una expresión de función inmediatamente invocada es un patrón el cual produce un alcance lexico usando usando Javascript. Alcance de función. Expresión de función imadiatamente invocada puede ser usada para evitar la elevación de las variables con los bloques, proteje enprotejiendo en contra contra la contaminación del ambien global y simultáneamente permite acceder a los métodos publicos al tiempo que conserva la privacidad en las variables definidas dentro de la función.

Este patrón ha sido referido como una función anónima autoejecutable, pero @cowboy (Ben Alman) introduce el termino IIFE como un término semántico más exacto para el patrón.

Fuente: Expresión de Función Inmediatamente Invocada (IIFE)

jQuery es una libreria en Javascript rápida, pequeña, y abundante en características. Contruida por John Resig.

Construido por Yahoo!, YUI es una libreria Javascript y CSS gratis, de código abierto para construir aplicaciones web altamente interactivas. El nuevo desarrollo ha sido detenido desde el 29 de Agosto del 2014.

Zepto es una libreria JavaScript minimalista para navegadores modernos con un API en gran parte compatible con jQuery. Si usas jQuery, estas listo para conocer como se usa Zepto.

  • Dojo Toolkit

Dojo son herramientas Javascript gratuitas, código abierto para construir aplicaciones web de alto rendimento. IBM y SitePen son patrocinantes el proyecto.

  • Underscore.js

Underscore.js es una libreria JavaScript que provee una maraña de ayudantes de programación funcional sin extender alguno de los objetos preconstruidos.

Construido por Google, Angular.js es como un polyfill para el HTML del futuro.

Cappuccino es un framework de código abierto que hace fácil la contrucción de aplicaciones del calibre de aplicaciones desktop que se ejecutan en un navegador web.

JavaScriptMVC es un framework de código abierto que con tiene las mejores ideas del desarrollo con jQuery.

Meteor es una plataforma de código abierto para construir aplicaciones web de alta calidad en una fracción de tiempo, siendo un desarrollador experto o quien solo acaba de comenzar.

Spice es un super mínimo (< 3k) y flexible framework MVC para javascript. Spice fue construido para agregar fácilmente a cualquier application existente y jugar bien con otras tecnologías tales como jQuery, pjax, turbolinks, node o cualquier itra que usted utilice.

Riot es un incriblemente rápido, poderoso y aún así pequeña libreria (MV*) del lado cliente para contruir aplicacione web de gran escala. A persar de su pequeño tamaño todos los bloques de construcción se encuentran: un motor de plantilla, enrutador, libreria de eventos y un patrón MVP estricto para mantener las cosas organizadas.

CanJS es un framework Javascript que hace el desarrollo de aplicaciones complejas sencillo y rápido. De fácil aprendizaje, pequeña, y sin ninguna asumpción de estructura de aplicación, pero con características modernas como etiquetar y vinculación doble vía.

CanJS is a JavaScript framework that makes developing complex applications simple and fast. Easy-to-learn, small, and unassuming of your application structure, but with modern features like custom tags and 2-way binding.
